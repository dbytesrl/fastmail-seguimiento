01 — Diagnóstico

Por qué hay que
refactorizar el código actual

Durante el análisis del sitio se detectaron problemas críticos que hacen que el código actual no sea seguro para continuar en producción.

Crítico
⚠️

Archivo malicioso activo (anons79.php)

Se encontró una webshell inyectada en el proyecto. Esto significa que un tercero tuvo o tiene acceso completo al servidor: puede leer archivos, ejecutar comandos, robar credenciales de base de datos y modificar cualquier contenido.

🔓

Versión de Laravel desactualizada

El framework corre en una versión sin soporte activo, sin parches de seguridad publicados desde hace años.

📦

Dependencias vulnerables

El composer.lock y el package.json tienen paquetes con vulnerabilidades conocidas y públicamente documentadas.

🔑

Autenticación débil

El sistema de tokens API no sigue los estándares modernos (JWT stateless), lo que expone sesiones a reutilización y robo.

🗂️

Sin separación de entornos

El código fuente, la lógica de negocio y los archivos generados (PDFs de cotizaciones) conviven en el mismo servidor sin aislamiento.

🛡️

Sin HTTPS forzado ni cabeceras de seguridad

El sitio no protege la transmisión de datos entre el usuario y el servidor.

No se trata de un sistema "que tiene bugs", sino de una instalación comprometida. Parcharlo parcialmente no resuelve el problema raíz. La recomendación es reescribir la aplicación sobre una base segura, aprovechando que toda la lógica de negocio ya fue mapeada y documentada.

02 — Solución

Qué se propone construir

Una aplicación web moderna en dos capas independientes, diseñada para escalar, segura desde su base y optimizada para los flujos reales de los instaladores.

Capa 1

Frontend SPA
(React)

  • Accesible desde cualquier navegador, celular o tablet, sin instalar nada
  • Deploy en servidor estático seguro (Vercel, Netlify o similar — costo casi cero)
  • Interfaz para diseñar proyectos, calcular materiales y generar cotizaciones en PDF
  • Motor de cálculo corriendo en el navegador: respuesta instantánea, sin depender de la velocidad del servidor

Capa 2

Backend API
(Laravel / Node.js)

  • Servidor dedicado para datos: usuarios, proyectos, historial de cotizaciones, catálogo y precios
  • Autenticación moderna con JWT
  • Acceso controlado: cada usuario solo ve sus propios proyectos
  • Historial completo: cada cotización guardada con fecha, versión de precios y estado
  • API documentada y separada — preparada para app mobile en el futuro

sólida y en producción

01 Mes

Base segura y lógica central

Fundaciones · Seguridad · Motor de cálculo

  • Montar entorno nuevo, limpio, sin la webshell ni el código comprometido
